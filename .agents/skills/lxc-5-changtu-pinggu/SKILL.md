# 离心创意场 ⑤ 打磨交付

> ①破题 → ②发散 → ③定创意核 → ④出大创意 → **⑤打磨交付**
> 把方案当整体审一遍，**润色到能直接给客户**，并产出 `成果稿.md`。

## 开工前
读工作区 `创意场记录.md` 和 ④ 起草的 `成果稿.md`（守则已自动加载）。前置：④已拍板 big idea / 文案。没拍板就停，回 `$lxc-control` / `$lxc-4-dingceng-tilian`。

## 怎么审（通读挑毛病，不打分表）
把方案读一遍，逐条点真问题、给修法（回哪一棒、怎么改）。常见病：
- **太平**——不够新鲜 → 回②再甩远一点。
- **太散**——核心不清 → 回③收一下。
- **不像品牌**——文案飘在外面、回不到产品 → 回④补品牌扣点。
- **不打动人**——只有聪明没有情绪 → 回④加情绪。
- **能看不能传**——缺让人想转发的钩子 → 回④强化双关 / 句式 / 玩法。
能小修就小修，别动不动推翻。

## 润色（终稿）
对主推方案做最后一道文字打磨：**双关 / 谐音优先**，口语顺口，砍多余字，去掉负面词，让每条都能直接念出口。

## 产出 `成果稿.md`（给人 / 客户看的）
按这个结构写，有文案感、不堆术语：
```md
# 〔项目〕创意方案

## 一句话洞察
（人群真正的心理）

## Big Idea
（一句话：这个 campaign 在玩什么）

## 主文案 / Slogan
（主句 + 3–5 条候选，标出主推）

## 系列延展
（同一机制套出的系列文案）

## KV / 执行方向
（画面怎么走、线上线下怎么落，给方向，不做细案）

## 为什么成立（两行）
（扣住哪个创意核、怎么回到品牌）
```

## 过关自检
- [ ] 真问题都点到了、给了"回哪棒怎么修"？
- [ ] 终稿文案**双关 / 谐音优先**、像人话、没负面词？
- [ ] `成果稿.md` 能**直接拿去给客户**吗？
- [ ] 没重新发散、没另起炉灶？

## 暂停 · 交人定
停下，请用户定：通过 / 还要改哪条 / 回哪一棒。

## 写进文件
定稿 `成果稿.md`；更新 `创意场记录.md`：⑤状态、审出的问题与修法、最终结论（通过 / 回炉到哪棒）、用户拍板。
