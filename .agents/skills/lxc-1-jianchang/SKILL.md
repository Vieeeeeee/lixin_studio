# 离心创意场 ① 命题建场（步骤1–3）

> 工作流位置：**①命题建场** → ②节点池与折线路径池 → ③结构识别 → ④顶层提拉 → ⑤场图评估与重构
> 本棒目标：把一个模糊命题，收束成「底座清楚 ＋ 1-2 个有重力的核心主张」。

## 开工前 · 读取项目记录
开始本棒前，先确认当前启动项目根目录。`创意场记录.md` 必须放在当前启动项目根目录，不放在「离心创意场」母文件夹。

然后检查当前启动项目根目录是否存在 `创意场记录.md`：
- **存在**：必须完整读取，确认当前阶段、已完成内容、用户已拍板内容、被否方向和下一步，再决定是否执行本棒。
- **不存在**：用 `references/record-template.md` 创建 `创意场记录.md`，写入当前阶段为本棒。
- 若记录显示**前置阶段未完成**：不得跳步，提醒用户先完成前置阶段。
- 若记录显示**本棒已完成**：不得重复执行，询问用户是继续下一棒、修改本棒还是重跑本棒。
- 创建或更新记录时，所有可空字段必须写“无”，信息缺失但未来需要补齐时写“待补”。

## 开工前 · 守则
本棒遵守母文件夹工作守则（已随项目自动加载，无需再读），尤其硬规则 1（先建场）、2（事实层具体）、10（每阶段暂停过关）。`离心创意场.md` 仅按需查段。
本棒**只做三件事**：收命题输入、扫抽象层、向心洞察。**不离心取点、不写文案、不画画面**——那是后面几棒的事。

## 动作一 · 收命题输入（步骤1）
先收齐这张**命题卡**：

| 项 | 内容 |
|---|---|
| 品牌 | |
| 产品 / 服务 | |
| **产品 / 服务的具体事实** | 成分、结构、功能、使用动作、价格、规格、包装、形态、流程… |
| 目标人群 | |
| **用户真实场景** | 什么时候用、在哪里用、和谁有关、前后发生什么 |
| 传播场景 | 节日 / 节点 / 货架 / 线上线下… |
| 商业目标 | 拉新 / 转化 / 声量 / 品效… |
| 时间节点 | |
| 平台环境 | |
| 必须传达的信息 | |
| 禁区 / 限制 | 不能碰的 |
| 已有品牌资产 | 符号 / 口号 / 代言 / 历史梗… |
| **当前传播资产 / 素材** | 图片、包装、代言人、门店、货架、社媒内容、过往传播战役 |
| **受众已有语言** | 用户评论、口头禅、搜索词、平台热词、真实吐槽 |
| 对手常规表达 / 旧打法 | 竞品惯用打法 |

**缺项规则：**
- **关键输入缺失 → 停**：`品牌 / 产品·服务 / 目标人群 / 商业目标 / 传播场景` 缺任一项，不准往下，一次问全、问清为止。
- **非关键项缺失 → 可标"待补"**，但必须注明它会卡住哪一棒。例如：`产品具体事实 / 用户真实场景 / 受众已有语言` 缺失 → 会制约 **②节点池与折线路径池**（事实层没料就打捞不出离心节点）；`当前传播资产` 缺失 → 会制约 **④顶层提拉 与 ⑤场图评估**。

## 动作二 · 扫描抽象层（步骤2）
画出**底座**，也画出**牢笼**。输出四张清单：

- **旧中心节点**：这个命题被大家默认怎么理解、怎么说？要具体到话术，别写"很老套"。
- **低势能 / 过度使用词**：正确但无聊、被用烂的词。
- **必须避开的表达**：落进去就死的旧路。
- **可留作底座的真实重力**：这些旧表达背后**真正有效**的情绪、需求或社会压力是什么？

> **关键校准**：不要只标记旧中心"老套"就全删掉。旧中心不是垃圾，它是底座——要避开的是旧**表达**，不是旧**命题**本身。必须拆出它为什么长期有效（如春节送礼"唯一·首选·C位"背后是"面子、人情、不能出错、要显得重视"）。核心主张不是凭空反对旧中心，而是**继承旧中心的真实重力，再换一个更新的表达结构**。

## 动作三 · 向心洞察（步骤3）
从命题里听暗流，收束出 **3–5 个核心主张候选**。每个候选必须填满下表——**缺"反对的旧表达"一栏的，不算合格洞察**（它可能还停在旧中心里）：

| 字段 | 说明 |
|---|---|
| 内部主张 | 一句话说清"我们到底要说什么"（是**重力源**，不是口号 / 最终文案） |
| 洞察来源 | 从哪条信息 / 矛盾里听出来的 |
| 反对的旧表达 | 它反对动作二里的哪个旧中心 |
| 继承的真实重力 | 它保留了旧中心背后的哪股真实需求 / 情绪 |
| 指向的新人群心理 | 戳中什么新的人群情绪 / 身份 |
| 情绪潜力 | 能升起什么情绪 |
| 商业相关性 | 和商业目标的关系 |
| 可展开性 | 后面能不能离心、能不能系列化 |
| 风险 | 哪里可能偏 / 散 / 俗 |

## 档位判定（轻 / 重档）
顺手判一下本命题量级，写进记录第 0 节「档位」字段，供后续各棒收放强度（见基座守则「轻档 / 重档」）：
- 轻档：单渠道 / 单条内容 / 小预算 / 快需求 / 非比稿。
- 重档：多渠道整合 / 比稿 / 大战役 / 系列化。
拿不准默认重档，并说明依据；用户可推翻。

## 过关自检（对照基座「命题建场」过关标准）
跑完逐条自检，不过关就回炉：
- [ ] 命题卡关键项是否齐全？非关键缺项是否已标"待补"并注明影响哪一棒？
- [ ] 旧中心是否列出且具体（到话术，不是泛指）？
- [ ] 是否拆出了旧中心的**真实重力**（为什么长期有效），而不是把旧中心当垃圾全删？
- [ ] **旧中心是否清楚？**（基座过关项）
- [ ] 每个主张候选是否都写明了"反对哪个旧表达"+"继承哪股真实重力"？
- [ ] **核心主张是否有重力？**（基座过关项——删掉它，方案是否坍塌？是重力源还是漂亮话？）

## 暂停 · 交人拍板
自检通过后**停下**，向用户汇报：①命题卡 ②抽象层四清单 ③3-5 个主张候选（带上表）。
请用户**选 1–2 个最有重力的核心主张**。AI 负责枚举与压缩，重力源由人来选。

## 本棒记录要求
本棒结束时，必须把以下内容写入 `创意场记录.md`（**禁止只在对话里汇报、不写入记录**）：
1. 阶段状态总表：①阶段产物是否存在 = 是；①用户拍板是否完成 = 等待用户；当前状态 = ①产物完成 / 等待用户拍板。
2. 当前状态：①命题建场已完成 / 等待用户选择核心主张；并在第 0 节填写「档位」（轻 / 重 + 一句依据）。
3. 命题卡（含待补项标记）。
4. 抽象层扫描：旧中心、低势能词、必须避开的表达、可留作底座的真实重力。
5. 3–5 个核心主张候选（含：反对的旧表达、继承的真实重力、洞察来源、情绪潜力、商业相关性、风险）。
6. 用户拍板记录 / ①核心主张拍板：已选择 = 无；被否 = 无；用户补充 = 无；状态 = 等待用户。
7. 下一步：请用户选择 1–2 个核心主张，进入 ②节点池与折线路径池。
8. 禁止重复：已扫过的旧中心、已否掉的主张方向；没有则写“无”。
