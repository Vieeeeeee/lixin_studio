# 离心创意场 ① 破题（拆 brief · 定原点）

> **①破题** → ②发散 → ③定创意核 → ④出创意小样 → ⑤打磨交付
> 这一棒只干一件事：把模糊 brief，收成 **8-10 个候选「创意原点」**，让用户锁 1-2 个当**主线**。

## 开工前
在启动项目根目录建 `创意场工作区/`（不和用户文件混），`创意场记录.md` 是进度入口：有就读、没有按 `references/record-template.md` 新建。守则已自动加载。

## 必做
1. **拆 brief**：收齐关键项——品牌 / 产品·服务 / 目标人群 / 商业目标 / 传播场景。缺关键项先问全；缺非关键项（产品具体事实、用户真实场景、受众口头禅 / 热词、已有资产、对手打法、时间 / 平台 / 禁区）标"待补"、注明会影响哪步。
2. **看清套路**：这命题大家默认怎么说？列出被用滥的表达。**别全删**——它背后那股真实需求 / 情绪（要面子、怕错过、想偷懒、想被看见…）要留作地基。
3. **给 8-10 个候选原点**：每个一两句说清——**从哪出发、地基是什么、接住了哪股真实需求、为什么有戏**。标出**推荐**。原点是"主线候选"，不是最终文案。

## 可做 · 看情况
- 想给某个原点配一句"走这条大概什么味道"的引子，可以——但别展开成方案（那是后面的事）。
- 标签（新鲜度 / 风险等）**需要才打**，不浪费就别打。

## 留余量
破题时若已蹦出某个完整好点子，**别压着**——记一笔（可先落成半张创意小样），但仍要把原点给齐，让用户先锁主线。

## 暂停 · 交人锁主线
停下，给用户：brief 要点、套路与背后的真实需求、**8-10 个候选原点（含推荐）**。请其锁 **1-2 个**当主线。

## 写进文件
更新工作区 `创意场记录.md`：①状态（产物=是 / 等待用户锁原点）、brief 要点、套路与真实需求、8-10 候选原点（含推荐）、用户锁定（先写"等待用户"）、下一步=进②发散。
