# 离心创意场 ① 破题（拆 brief · 找洞察 · 定主张）

> **①破题** → ②发散 → ③定创意核 → ④出大创意 → ⑤打磨交付
> 这一棒把模糊 brief，收成「套路看清 ＋ 1–2 个有重力的核心主张」。

## 开工前
在启动项目根目录建 `创意场工作区/`（不和用户文件混），其中 `创意场记录.md` 是进度入口：已有就读它定位进度，没有就按 `references/record-template.md` 新建。守则已自动加载；`离心创意场.md` 仅按需查段。空字段写"无"、缺待补写"待补"。
本棒只做三件事：收 brief、看清套路、定主张。**不发散、不写文案。**

## 动作一 · 收 brief
收齐这些（**关键项缺就停下、一次问全**；非关键缺标"待补"并注明会卡哪棒）：
- 关键：品牌 / 产品·服务 / 目标人群 / 商业目标 / 传播场景
- 重要（缺会拖累后面）：产品具体事实、用户真实场景、受众已有语言（评论 / 口头禅 / 热词）、已有品牌资产、对手常规打法、时间 / 平台 / 必传信息 / 禁区

## 动作二 · 看清套路
这命题大家**默认怎么说**？列出被用滥的套路表达（具体到话术，别写"老套"），标出**必须避开**的。
**关键**：套路别全删——拆出它**为什么长期有效**（背后的真实需求 / 情绪 / 社会压力），这股劲留作底座。核心主张不是凭空反对套路，而是**接住这股真实需求、换个更新的说法**。

## 动作三 · 定核心主张
收出 **3–5 个核心主张候选**，每个写清：一句话主张（内部用、不是文案）／ 它反对哪个套路 ／ 接住哪股真实需求 ／ 戳中什么新人群心理 ／ 商业相关 ／ 风险。
**没写清"反对哪个套路"的不算数**——它可能还在套路里。

## 顺手定档
轻档（单条 / 单渠道 / 快需求 / 非比稿）还是重档（多渠道 / 比稿 / 大战役 / 系列）？写进记录，轻档后面各棒收着做；拿不准默认重档。

## 过关自检
- [ ] 关键 brief 齐了？缺的标"待补"并注明卡哪棒？
- [ ] 套路看清且具体？拆出了背后的真实需求（没当垃圾全删）？
- [ ] 每个主张都写了"反对哪个套路 + 接住哪股需求"？
- [ ] 主张有重力（删了就塌、是重力源不是漂亮话）？

## 暂停 · 交人选
停下，给用户：brief 要点、套路与真实需求、3–5 个主张候选。请其选 **1–2 个最有重力的**。AI 收集压缩，重力源由人定。

## 写进文件
新建 / 更新工作区 `创意场记录.md`（禁止只在对话说、不写记录）：①状态（产物=是 / 等待用户拍板）、档位、brief 要点、套路与真实需求、3–5 候选、用户选定（先写"等待用户"）、下一步=进②发散、不要再做的（无则"无"）。
